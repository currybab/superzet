# Security Policy

## Supported Releases

Security fixes are currently targeted at the latest `main` branch and the latest public preview release.

## Reporting a Vulnerability

Do not open a public issue for a security vulnerability.

If GitHub private vulnerability reporting is enabled for this repository, use it. Otherwise contact the maintainers privately through GitHub and include:

- affected version or commit
- reproduction details
- impact assessment
- any suggested mitigation

We will acknowledge reports as quickly as practical and coordinate disclosure once a fix or mitigation is ready.
