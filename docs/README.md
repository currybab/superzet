# superzet Docs

Welcome to `superzet`'s documentation.

To preview the docs locally, install [mdBook](https://rust-lang.github.io/mdBook/) (`cargo install mdbook@0.4.40`) and run:

```sh
mdbook serve docs
```

Before committing, verify that the docs are formatted in the way Prettier expects with:

```sh
cd docs && pnpm dlx prettier@3.5.0 . --write && cd ..
```

## Preprocessor

The docs use a custom mdBook preprocessor that lives in `crates/docs_preprocessor`.

If you need to bypass it locally, comment out `[preprocessor.zed_docs_preprocessor]` in `book.toml`.

## Images and videos

To add images or videos to the docs, upload them elsewhere and link out to them from the docs.

Putting binary assets directly in the repository will bloat the repo over time.

## Table of Contents

The table of contents files (`theme/page-toc.js` and `theme/page-doc.css`) were initially generated by [`mdbook-pagetoc`](https://crates.io/crates/mdbook-pagetoc).

Since all this preprocessor does is generate static assets, we do not need to keep it around once they have been generated.

## Referencing Keybindings and Actions

### Keybindings

`{#kb scope::Action}` - e.g. `{#kb zed::OpenSettings}`.

This renders a code element like `<code>Cmd + , | Ctrl + ,</code>`. A client-side plugin then shows the actual keybinding for the current platform.

### Actions

`{#action scope::Action}` - e.g. `{#action zed::OpenSettings}`.

This renders a readable action label like `zed: open settings`.

### Creating New Templates

Templates are functions that modify docs source before rendering, usually with a regex replace.

See `crates/docs_preprocessor/src/main.rs` for the current action and keybinding templates.

### References

- Template trait: `crates/docs_preprocessor/src/templates.rs`
- Example template: `crates/docs_preprocessor/src/templates/keybinding.rs`
- Client-side plugins: `docs/theme/plugins.js`

## Postprocessor

The postprocessor is implemented as a sub-command of `docs_preprocessor`. It wraps the built-in HTML renderer so page-specific titles and descriptions can be filled from front matter.

Example:

```md
---
title: Some more detailed title for this page
description: A page-specific description
---

# Editor
```

If no front matter is present, `title` and `description` fall back to the defaults in `book.toml`.

### Known limitations

- Double quotes and multi-line values are not supported in front matter.
- Front matter must be at the top of the file.
- `title` and `description` are not HTML escaped, so keep them simple ASCII text.
